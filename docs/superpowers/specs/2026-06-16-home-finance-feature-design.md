# Home Finance 功能设计

日期：2026-06-16

## 目标

Home Finance 第一版面向多人家庭支出协作。应用采用自托管 Go API + SQLite 后端，React/Tauri 设备端连接自托管服务。第一版重点验证“创建家庭账本、成员通过邀请码加入、共同记录支出、查看基础分析”的主流程。

## 已确认范围

- 多设备协作为主。
- 自托管服务：用户自行部署 Go API，设备端配置服务地址。
- 成员加入方式：家庭邀请码 + 昵称。
- 家庭账目全员可见。
- 第一版只记录支出，不记录收入、结余和现金流。
- 分析能力为基础统计：月度总支出、分类占比、成员金额/笔数、最近记录。
- 产品面拆为设备端和轻量管理后台。

## 非目标

- 不做手机号、邮箱或第三方登录。
- 不做私密账目或按角色控制账目可见性。
- 不做预算、超支提醒、趋势预测或异常消费检测。
- 不做离线写入队列和冲突合并。
- 不做复杂 SaaS 多租户运维后台。
- 不做自动备份调度。

## 产品入口

### 设备端

设备端给家庭成员在手机或桌面日常使用，强调低摩擦记账和查看分析。

核心能力：

- 配置自托管 API 地址并检查连接状态。
- 输入邀请码和昵称加入家庭。
- 保存本设备成员会话，下次直接进入家庭账本。
- 查看首页总览。
- 新增、编辑、删除支出。
- 按月份、分类、成员筛选支出。
- 查看基础月度分析。
- 管理当前连接、家庭会话和退出登录。

### 管理后台

管理后台同时覆盖家庭管理和轻量服务运维。第一版保持必要功能，不扩展成复杂 SaaS 控制台。

核心能力：

- 管理员口令登录。
- 查看 API 状态、版本、数据库路径和基础数据量。
- 创建和编辑家庭。
- 生成、复制、停用邀请码。
- 查看、重命名、停用成员。
- 维护支出分类。
- 按家庭和月份导出支出 CSV。
- 展示 SQLite 数据库路径和手动备份说明。

## 数据模型

### households

家庭账本。

字段：

- `id`
- `name`
- `status`
- `created_at`
- `updated_at`

### members

家庭成员。第一版成员不是正式账号，只代表通过邀请码加入的成员身份。

字段：

- `id`
- `household_id`
- `nickname`
- `session_token_hash`
- `status`
- `created_at`
- `last_active_at`

### invite_codes

家庭邀请码。后端只保存邀请码哈希。

字段：

- `id`
- `household_id`
- `code_hash`
- `status`
- `expires_at`
- `usage_count`
- `created_at`

### categories

支出分类。

字段：

- `id`
- `household_id`
- `name`
- `color`
- `sort_order`
- `status`
- `created_at`
- `updated_at`

默认分类：

- 餐饮
- 购物
- 交通
- 住房
- 医疗
- 教育
- 娱乐
- 其他

### expenses

支出记录。金额用整数分保存，避免浮点误差。

字段：

- `id`
- `household_id`
- `member_id`
- `category_id`
- `amount_cents`
- `currency`
- `spent_at`
- `note`
- `deleted_at`
- `created_at`
- `updated_at`

### admin_sessions

管理后台会话。第一版使用服务器初始化口令换取短期管理 token。

字段：

- `id`
- `token_hash`
- `expires_at`
- `created_at`

## 同步和鉴权

- 后端 SQLite 是权威数据源。
- 设备端每次进入页面或手动刷新时拉取最新数据。
- 新增、编辑、删除支出必须在线提交。
- 第一版不做离线写入队列。
- `/api/*` 使用设备会话 token。
- `/admin/*` 使用管理 token。
- 邀请码只用于加入家庭，不用于长期请求。

## API 设计

### 通用

- `GET /health`
  - 检查服务可用性。

### 设备端 API

- `POST /api/join`
  - 使用邀请码和昵称加入家庭。
  - 返回家庭信息、成员信息和设备会话 token。

- `GET /api/me`
  - 根据设备会话 token 返回当前家庭和成员信息。

- `GET /api/expenses?month=YYYY-MM&categoryId=&memberId=`
  - 获取家庭支出列表，默认当前月。

- `POST /api/expenses`
  - 新增支出。
  - 字段：金额、分类、日期、备注。

- `PATCH /api/expenses/:id`
  - 编辑支出。
  - 允许修改金额、分类、日期、备注。

- `DELETE /api/expenses/:id`
  - 软删除支出。

- `GET /api/categories`
  - 获取可用支出分类。

- `GET /api/analytics/monthly?month=YYYY-MM`
  - 获取本月总额、分类占比、成员金额/笔数和最近记录。

### 管理后台 API

- `POST /admin/login`
  - 管理员口令登录，返回管理 token。

- `GET /admin/status`
  - 返回服务状态、版本、数据库路径、家庭数量、成员数量、支出数量。

- `GET /admin/households`
- `POST /admin/households`
- `PATCH /admin/households/:id`

- `POST /admin/households/:id/invite-codes`
  - 生成新邀请码。

- `PATCH /admin/invite-codes/:id`
  - 停用或刷新邀请码。

- `GET /admin/households/:id/members`
- `PATCH /admin/members/:id`
  - 重命名或停用成员。

- `GET /admin/households/:id/categories`
- `POST /admin/households/:id/categories`
- `PATCH /admin/categories/:id`

- `GET /admin/exports/expenses.csv?householdId=&month=`
  - 导出支出 CSV。

## 设备端页面

### 连接服务页

- 输入自托管 API 地址。
- 调用 `/health` 检查服务。
- 成功后保存服务地址到本地。
- 失败时提示地址格式错误、连接失败或服务不可用。

### 加入家庭页

- 输入邀请码和昵称。
- 调用 `POST /api/join`。
- 成功后保存设备会话 token。
- 已加入设备下次直接进入首页。

### 首页总览

- 显示家庭名称、当前月份、同步状态。
- 指标：本月总支出、支出笔数、最高分类、最近一次记录。
- 展示最近支出列表。
- 主操作：新增支出。

### 新增/编辑支出页

- 字段：金额、分类、消费日期、备注。
- 默认日期为今天。
- 默认币种为 CNY。
- 编辑时显示原始记录。
- 删除支出需要二次确认。

### 支出列表页

- 默认按月份倒序。
- 支持按月份、分类、成员筛选。
- 点击记录进入编辑。
- 空状态提示新增第一笔支出。

### 分析页

- 月度总支出。
- 分类占比列表。
- 成员金额和笔数排行。
- 最近记录。
- 第一版使用列表和简单条形占比，不要求复杂图表库。

### 设置页

- 当前服务地址。
- 当前家庭和成员昵称。
- 重新连接服务。
- 退出家庭会话。

导航：设备端使用底部导航，总览、支出、分析、设置。

## 管理后台页面

### 管理员登录页

- 输入管理员口令。
- 登录成功后保存管理 token。

### 系统状态页

- API 状态、版本、数据库路径。
- 家庭数量、成员数量、支出数量。
- 简单运行状态，不做复杂监控。

### 家庭管理页

- 创建家庭。
- 编辑家庭名称。
- 查看家庭列表。

### 邀请码页

- 为指定家庭生成邀请码。
- 查看有效、停用、过期状态。
- 停用邀请码。
- 复制邀请码。

### 成员管理页

- 查看成员列表和最后活跃时间。
- 重命名成员。
- 停用成员。

### 分类管理页

- 新增、重命名、停用分类。
- 设置颜色和排序。
- 停用分类不影响历史支出显示。

### 数据导出页

- 按家庭和月份导出 CSV。
- 显示 SQLite 数据库路径和手动备份说明。

导航：管理后台使用侧边导航，状态、家庭、邀请码、成员、分类、导出。

## 错误处理

- 连接失败：设备端提示服务地址不可用。
- token 失效：设备端回到加入家庭页，后台回到管理员登录页。
- 邀请码无效或过期：提示重新获取邀请码。
- 成员停用：设备端退出当前家庭会话。
- 分类停用：历史支出仍显示分类名称，但不能再用于新支出。
- 支出金额无效：提示金额必须大于 0。
- API 返回未知错误：展示通用错误提示，并保留重试入口。

## 测试设计

### 后端测试

- 邀请码加入家庭：有效、无效、过期、停用。
- 设备会话鉴权：缺失、无效、成员停用。
- 支出 CRUD：新增、编辑、软删除、金额必须大于 0。
- 支出查询：按月份、分类、成员筛选。
- 分类停用：历史支出可读，新支出不可用。
- 月度分析：总额、分类占比、成员金额/笔数。
- 管理后台鉴权：无 token、错误 token、有效 token。
- CSV 导出：字段顺序、金额格式、月份筛选。

### 前端测试

- 服务地址校验和连接状态。
- 加入家庭表单校验。
- 支出表单金额、日期、分类校验。
- API 错误状态展示。
- 分析数据格式化。
- 会话 token 保存、退出和失效处理。

- 成员列表、重命名、停用。

### 阶段 3：分类与支出记录

- 分类默认初始化和管理。
- 支出新增、编辑、删除。

### 阶段 4：基础分析与导出

- 月度统计接口。
- 设备端分析页。
- CSV 导出。
- README 更新部署和使用流程。

### 阶段 5：体验打磨

- 移动端布局优化。
- API 错误统一展示。
- 空状态、加载状态、网络失败提示。
- Tauri/Android 配置补齐。

## 验收标准

- 管理员可以登录后台，创建家庭并生成邀请码。
- 家庭成员可以在设备端配置服务地址，用邀请码和昵称加入家庭。
- 成员可以新增、编辑、删除支出。
- 所有家庭成员可以看到同一家庭的全部支出。
- 设备端可以按月查看总支出、分类占比、成员金额/笔数和最近记录。
- 管理后台可以维护成员、分类和邀请码。
- 管理后台可以导出指定家庭和月份的支出 CSV。
- 后端核心 API 有单元测试或处理器测试覆盖。
- 前端关键表单和数据格式化逻辑有单元测试覆盖。
